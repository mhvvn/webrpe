# RPE Smart Portal

A comprehensive web portal for the Energy Power Generation Engineering (RPE) study program at Politeknik Negeri Batam. This application serves as an information center for students, lecturers, and the public, as well as an administrative dashboard for managing content.

## Tech Stack

*   **Frontend**: React, TypeScript, Tailwind CSS, Vite
*   **Backend**: Node.js, Express
*   **Database**: PostgreSQL
*   **Authentication**: JWT (JSON Web Tokens)

---

## 🚀 How to Run Locally

Follow these steps to set up and run the project on your local machine.

### 1. Prerequisites

Ensure you have the following installed:
*   [Node.js](https://nodejs.org/) (v16 or higher)
*   [PostgreSQL](https://www.postgresql.org/)

### 2. Database Setup

1.  Open your PostgreSQL tool (pgAdmin, DBeaver, or terminal).
2.  Create a new database named `rpe_smart_portal` (or another name if you prefer).
3.  Execute the SQL script found in `backend/database_schema.sql` to create the necessary tables and seed initial data.
    *   *Note: This script requires the `uuid-ossp` extension, which is included in the script.*

### 3. Application Setup

1.  **Clone/Open the project** in your terminal.
2.  **Install dependencies**:
    ```bash
    npm install
    ```
3.  **Environment Configuration**:
    *   There is a `.env` file in the root directory. Open it and update the database credentials to match your local PostgreSQL setup:
    ```env
    DB_USER=your_postgres_username
    DB_PASSWORD=your_postgres_password
    DB_HOST=localhost
    DB_PORT=5432
    DB_NAME=rpe_smart_portal
    JWT_SECRET=any_secret_string_for_development
    PORT=3000
    ```

### 4. Running the Application

You can run the application in **Development Mode** (for editing code) or **Production Mode** (simulating the final deployment).

#### Development Mode (Recommended for Editing)

Runs the backend and frontend separately with hot-reloading.

1.  **Terminal 1 (Backend):**
    Start the Express API server.
    ```bash
    npm start
    ```
    *(Runs `node backend/server.cjs`)*
    *The server runs on `http://localhost:3000`.*

2.  **Terminal 2 (Frontend):**
    Start the Vite development server.
    ```bash
    npm run dev
    ```
    *The frontend runs on `http://localhost:5173`.*

> **Note:** In this mode, the frontend (`:5173`) communicates with the backend (`:3000`) via API calls.

#### Production Mode (Simulate Deployment)

Builds the frontend and serves it through the Node.js backend. User only needs to visit one URL.

1.  **Build the Frontend:**
    ```bash
    npm run build
    ```
    *This compiles the React app into the `dist` folder.*

2.  **Run the Server:**
    ```bash
    npm start
    ```

3.  **Visit the App:**
    Open `http://localhost:3000` in your browser. The backend now serves the frontend files.

### 5. Accessing the App

*   **Public Portal**: Visit the URL provided by Vite (e.g., `http://localhost:5173`).
*   **Admin Dashboard**: Navigate to `/admin/login`.
    *   **Default Admin Credentials** (from seed data):
        *   Username: `superadmin`
        *   Password: `123`
        *   Username: `dosen`
        *   Password: `123`

## 📁 Project Structure

```
rpe-smart-portal/
├── src/                  <-- Frontend Code (React)
│   ├── components/       # Reusable UI components
│   ├── context/          # Global state (User, News, etc.)
│   ├── pages/            # Page views (Public & Admin)
│   ├── services/         # API integration
│   ├── App.tsx           # Main App component
│   └── main.tsx          # Entry point
│
├── backend/              <-- Backend Code (Server)
│   ├── server.cjs        # Express Server Entry Point
│   ├── db.cjs            # Database Connection
│   └── database_schema.sql # Database Schema & Seed Data
│
├── public/               <-- Static Assets (Images, Icons)
├── dist/                 <-- Build Output (Generated by npm run build)
├── .env                  <-- Environment Variables
├── package.json
└── README.md
```
